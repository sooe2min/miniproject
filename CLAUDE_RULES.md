# 클로드 개발 지침 (Claude Rules)

## 📌 1. 필수 개발 원칙 (바이브 코딩 엄격 금지)

- **초보자 디버깅 최적화:** 작성하는 모든 코드는 초보자도 쉽게 읽고 디버깅할 수 있도록 구조적이고 직관적이어야 합니다. 감으로 때려 맞추는 코딩(바이브 코딩)을 엄격히 금지합니다.
- **기술 스택 제한:** Tailwind CSS, Bootstrap 등 어떠한 외부 스타일 라이브러리 및 UI 프레임워크도 **절대 사용하지 마세요.**
- **순수 기술 지향:** 오직 순수 HTML, 순수 CSS, 순수 JavaScript (Vanilla JS)로만 화면과 기능을 구현해야 합니다.

## 🎨 2. 디자인 및 CSS 규칙

- **디자인 무드:** 토스(Toss) 및 노션(Notion) 스타일의 극도로 미니멀하고 세련된 모던 라이트 모드 웹 디자인을 구현하세요.
- **테마 및 색상 관리:** 생성하는 CSS 파일 맨 위에 `:root` 변수를 선언하여 색상을 통일성 있게 관리하세요. (메인 포인트 컬러: 세일즈포스 블루 `#00A1E0` 필수 사용)
- **레이아웃 정렬 규칙:** `float`나 고정 `position` 사용을 금지합니다. 초보자가 직관적으로 이해하고 수정하기 쉬운 **Flexbox 레이아웃** 위주로만 화면을 정렬하세요.
- **가독성 주석:** CSS 파일 내의 정렬 및 주요 배치 코드 옆에는 해당 스타일이 어떤 역할을 하는지 친절한 한 줄 주석을 반드시 달아주세요.

## 🏗️ 3. 컴포넌트 파일 분리 및 HTML 자동 연결 규칙 (중요)

- **HTML 레이아웃 유지:** 제공된 메인 HTML(`detail.html`)의 큰 틀과 기존 클래스명 구조(예: `class="job-postings-card-list"` 등)는 **절대 수정하거나 접두사를 붙여서 변경하지 마세요.** 오직 `<!-- [팀원 가이드] -->` 주석이 있는 내부 영역만 채워야 합니다.
- **신규 요소 네이밍 접두사(Prefix) 필수 적용:** 주석 내부에 **'당신이 새로 추가하는'** HTML 태그의 클래스명과 자바스크립트 변수/함수명에는 충돌 방지용 유니크 접두사를 붙이세요. (기존 HTML 구조는 유지하고, 새로 생성하는 컴포넌트 내부 요소에만 적용)
  - _예시 (새로 만드는 요소만):_ `.sf-gb-card`, `function sfGbFetch()`
- **독립 파일 생성 (모듈화):** 요청받은 컴포넌트의 CSS와 JS는 반드시 독립된 외부 파일로 분리하여 작성해야 합니다. HTML 파일 내부에 스타일 코드를 직접 넣는 `<style>` 태그나, 자바스크립트 로직을 직접 짜 넣는 내장형 `<script>` 태그 삽입을 엄격히 금지합니다. 오직 파일 연결을 위한 `<link href="...">`와 `<script src="...">` 태그 형태만 허용합니다.
- **HTML 연결 가이드 필수 제공 (최종 출력 규칙):** 코드를 모두 작성한 후, 사용자가 이 컴포넌트를 메인 HTML에 바로 조립할 수 있도록 `detail.html`에 추가해야 할 `<link>` 태그와 `<script>` 태그의 정확한 삽입 위치와 코드를 안내 문구와 함께 최종 답변 하단에 반드시 포함해서 출력하세요.
  - _예시:_ `<!-- </head> 바로 위에 추가 --> <link rel="stylesheet" href="./guestbook.css">`
  - _예시:_ `<!-- </body> 바로 위에 추가 --> <script src="./guestbook.js"></script>`

## ⚡ 4. 자바스크립트 및 데이터(API) 통신 규칙

- **함수 네이밍 기본 규칙:** 다른 기능과 이름이 겹치지 않도록, 당신이 만드는 모든 함수 이름 앞에는 무조건 유니크한 접두사(Prefix)를 붙여서 일반 함수 형태로 작성하세요.
  - _Good:_ `function sfGbFetchData() { ... }`
  - _Bad:_ `function fetchData() { ... }`
- **DOM 선택 및 id 네이밍 규칙:** 자바스크립트에서 이벤트를 걸거나 태그를 선택할 때는 가장 직관적인 **`id` 속성**을 활용하세요. 이때 id 이름 맨 앞에도 당신만의 유니크한 접두사(Prefix)를 반드시 붙여야 합니다.
  - _Good (HTML):_ `<button id="sf-gb-submit-btn">등록</button>`
  - _Good (JS):_ `const submitBtn = document.getElementById('sf-gb-submit-btn');`
  - _Bad (접두사 없음):_ `<button id="submit-btn">등록</button>`
- **백엔드 통신 기본:** 본 프로젝트는 로컬에서 `json-server`를 활용해 가상 REST API 서버(Port: 3000)를 구동합니다. 서버에서 데이터를 가져오거나 보낼 때는 순수 `fetch()` 문법을 사용해 구현하세요. (이때 가독성이 좋은 `async` / `await` 패턴을 적극 활용합니다.)
- **상세한 한글 주석:** 이벤트 리스너와 비동기 통신(`fetch`) 부에는 동작 과정과 역할을 초보자가 쉽게 이해할 수 있도록 한글 주석을 상세히 달아주세요.
